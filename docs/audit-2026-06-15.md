# Editor Project — Consolidated Audit (2026-06-15)

**Scope:** 404 TS files (`src/`), 247 Python files (`backend/`), shaders, workers, shared schemas, electron, build config.
**Method:** 8 parallel Explore agents, one per slice; results cross-referenced + deduped.
**Totals:** ~165 distinct findings → 14 Critical, 26 High, 49 Medium, 76 Low.

Severity rubric:
- **Critical** — can crash, corrupt, leak unbounded, or break SSoT.
- **High** — wrong results / strong perf cliff / large-scale duplication.
- **Medium** — edge-case bug or noticeable perf.
- **Low** — polish / hygiene / micro-opt.

Status legend:
- `[x]` resolved (commit SHA noted)
- `[~]` partially resolved (notes inline)
- `[ ]` open

**Progress snapshot:** 14 Critical → 12 fully resolved, 2 partial, 0 open. 26 High → 13 resolved (13 open). Medium & Low buckets largely open; a handful of Low items landed in the mechanical wave.

## 🟥 CRITICAL — fix before further feature work

- [x] **C1. SSE EventBus has unbounded queues + leaks queues on hard disconnect** — resolved in `83a3668`
  - `backend/app/state/events.py:20` — `asyncio.Queue()` created with **no `maxsize`**. Long sessions or chatty publish paths grow memory without bound; `put_nowait` never backpressures.
  - `backend/app/state/events.py:16, 25-29` — Queue refs leak in `_queues[session_id]` if a client dies without graceful close (kill -9, network blip). Slow leak per reconnect.
  - `backend/app/api/state.py:199-221` — cleanup only runs if the async generator is GC'd; not guaranteed.
  - **Fix landed:** `maxsize=1000`; on QueueFull drain + inject synthetic `state.gap`; empty subscriber buckets pruned on unsubscribe.

- [x] **C2. Persistence writes per-node image data into every checkpoint** — resolved in per-node cluster (`e208889`)
  - `backend/app/session/persistence.py:36-39` — `_EXCLUDE_FROM_PERSIST` only excludes the *singleton* `image_bytes` / `prepare_result`, **not** `image_bytes_by_node` or `prepare_result_by_node`. Multi-image-node sessions balloon the JSON by MB per checkpoint.
  - **Fix landed:** `_EXCLUDE_FROM_PERSIST` extended with `image_bytes_by_node`, `mime_type_by_node`, `prepare_result_by_node`.

- [x] **C3. Undo/redo silently drops per-node image context** — resolved in per-node cluster (`831c7d0`, `d9a7a63`)
  - `backend/app/session/history.py:36-69` — `Snapshot.capture()` predates the per-node refactor; doesn't capture `image_context_by_node` or `prepare_result_by_node`.
  - `backend/app/state/document.py:94-122` — `apply_snapshot()` restores canonical/widgets/masks but ignores the same per-node fields → undo wipes analysis context on multi-image docs.
  - **Fix landed:** `Snapshot` carries `image_context_by_node`; `apply_snapshot` restores it + clears the legacy singleton; `history.applied` event payload extended with `imageContextByNode`. Frontend consumer side is a separate one-line follow-up.

- [x] **C4. Synchronous Anthropic call on the asyncio loop** — resolved in `83a3668`
  - `backend/app/api/analyze.py:386` — REST `/analyze` calls `client.analyze_image()` (sync, 30–120 s) without `run_in_executor` / `asyncio.to_thread`. **Blocks the entire event loop** — all SSE, tool calls, health checks freeze for the duration.
  - `backend/app/services/sam_client.py:42-46` — SAM model load (`SAM2ImagePredictor.from_pretrained`, multi-GB) also runs sync inside an async handler on first use.
  - **Fix landed:** `/analyze` + `/name-region` route the sync Anthropic + SAM work through `asyncio.to_thread`. The lazy SAM model load on first call is still sync inside the worker thread; a follow-up could eager-load at startup.

- [x] **C5. REST tools endpoint has zero rate limiting** — resolved in `83a3668`
  - `backend/app/api/tools_rest.py:17-23` — `/api/tools/{name}` bypasses the 30/min throttle that the MCP path enforces (`mcp/server.py:93`). Cheap DoS or expensive-tool hammering.
  - **Fix landed:** moved `_rate_limiter` to `deps.get_tool_rate_limiter()`; REST + MCP share one bucket per session; REST returns 429 on overflow.

- [x] **C6. Sessions on disk never pruned** — resolved in `83a3668`
  - `backend/app/services/session_store.py:195-201` — `prune_disk()` defined but **never called anywhere** in the codebase. Sessions accumulate forever (with up to 2 MB image bytes each).
  - **Fix landed:** `main.py` lifespan task runs `prune_disk()` every `disk_prune_interval_s` (default 1 h) for entries older than `disk_session_max_age_s` (default 7 d).

- [~] **C7. setState-during-render in two components** — partial resolution in `977bedc`
  - [x] `src/components/inspector/adjustments/HslPanelView.tsx:38-40` — `setBand(bands[0].key)` in render. **Fix landed:** replaced with a derived `activeBand`.
  - [~] `src/components/workspace/ImageNode.tsx:87-90` — `setPrevChromeVisible` + `setCompareHeld` in render body. **Kept as the documented previous-prop pattern** (React docs explicitly endorse this; the `useEffect` form trips the cascading-renders ESLint rule). Comment expanded to make the choice explicit.
  - [~] `src/components/CommandPalette.tsx:180-184` — same pattern, kept for the same reason. No code change.

- [~] **C8. Stale-closure / cross-store mutation hazards in SSE event handler** — partially resolved
  - [x] `src/store/backend-state-slice.ts:243` — `state.gap` handler captures `sid` then fires `useBackendState.getState().setSnapshot()` in a fire-and-forget async closure. No re-check that the session is still active → wrong-session writes after a reset. **Fix landed (`8d92942`):** the closure now re-checks `useBackendState.getState().sessionId === sid` before writing; logs and drops on mismatch. Regression test asserts the late refetch is dropped when sessionId changes mid-fetch.
  - [x] `src/store/backend-state-slice.ts:323-325` — Inside an Immer producer, calls `useEditorStore.getState().consumePinRequest()` / `setPinnedWidgetParams()`. Cross-store mutation from a state reducer; if either store resets mid-event, state diverges. **Fix landed (`8d92942`):** `applyEvent` now uses a side-effects-queue pattern — cross-store calls and `tetherWorkspaceWidget` are pushed to a local array inside the Immer producer and drained AFTER `set(...)` returns. The H4 cluster's `useSuggestionsUi.markPending` bridge was moved to the same queue.
  - [~] `src/core/document.ts:243-246` — `openImage()` triggers `useAiSession.getState().reset() / openSession()` in a deferred microtask after committing user-visible state. Open-image-then-open-image races. **Partial fix landed (`3cf57f3`):** hoisted the dynamic `import('@/hooks/useImageContext')` to a static top-of-file import; eliminates the import-resolution race window. - [x] **C9. Two SSE/REST race windows because the document write-lock isn't held** — resolved
  - [x] `backend/app/api/state.py:60-66` — `/state/{sid}` computes a snapshot **without** `store.with_document_lock(sid)`. Torn read possible while a tool is mid-mutation. **Fix landed (`09f935d`):** `state_snapshot` now wraps `compute_snapshot(doc)` in `with _store().with_document_lock(sid) as doc:`. Regression test asserts the lock is acquired.
  - [x] `backend/app/api/state.py:176-198` — initial replay events for SSE captured outside the write lock; the inline comment claims otherwise. **Fix landed (`09f935d`):** `state_events` now wraps both the `bus.subscribe(sid)` call and the `doc.history` replay capture in the document write lock — atomic with respect to any concurrent mutator. Inline comment rewritten to reflect actual behaviour. Regression test asserts the lock is acquired.
  - Bonus: `get_mask_bytes` had the same pattern (read `doc.masks` outside the lock). Wrapped while we were here (`09f935d`); regression test included.

- [x] **C10. Race + lost-update on compound widget edits** — resolved
  - [x] `backend/app/tools/widgets/set_widget_param.py:82-83` — two concurrent set-param calls on different keys of the same compound widget both read empty `locked_params`, both append, second overwrites first. Lock state lost. **Audit framing didn't apply:** `set_widget_param` is `kind="mutate"`, so the tool registry already serialises calls per session via `with_document_lock` (`tools/registry.py:117`). Two concurrent calls on the same session queue; no in-process race. Documented in the handler's doctrine comment (`2744956`) so future readers don't reintroduce the concern.
  - [x] Same file 44-55: if `node` is None (binding exists, target node deleted), the canonical write is silently skipped → widget vs. op-graph divergence. **Fix landed (`2744956`):** now raises `_OrphanBinding` BEFORE any state mutation; mapped to a new `orphan_binding` error envelope code. Tests at `backend/tests/tools/widgets/test_set_widget_param_orphan.py` cover (a) the raise, (b) the state-invariant that no binding/canonical mutation occurs on the orphan path, and (c) the happy path still works (`83c93b6`).

- [x] **C11. Premultiplied-alpha / luma-coeff drift in WebGL pipeline** — resolved in `b48dea8`
  - `src/shaders/split-tone.glsl.ts:27` — uses BT.601 `(0.299, 0.587, 0.114)` luma; every other shader (basic-adjustments, clarity, sharpen, histogram-compute, lut-parser) uses BT.709 `(0.2126, 0.7152, 0.0722)`. Visible threshold drift on saturated images.
  - **Fix landed:** split-tone now uses BT.709 too.

- [x] **C12. Layer compositor recurses without a cycle guard** — resolved in `b48dea8`
  - `src/lib/layer-compositor.ts:68-113` — `renderLayer()` follows `parentLayerId` recursively. Malformed/self-referencing tree → infinite recursion → tab crash. No depth cap, no `Set<seen>`.
  - **Fix landed:** `renderLayer(layer, seen = new Set())` bails with a console warning when a cycle is detected.

- [x] **C13. Frontend reloads source texture every frame** — resolved in `b48dea8`
  - `src/shaders/pipeline.ts:416-432` — `setSource()` unconditionally `gl.deleteTexture` + `texImage2D`. On a 4K image, that's ~100 MB/frame re-upload even when only an adjustment param moved. Dominates GPU upload time.
  - **Fix landed:** `setSource(source, dirty=true)` reuses the same `WebGLTexture`; skips upload entirely on `dirty:false` + same identity.

- [x] **C14. CORS default leaks dev origins to prod** — resolved in `83a3668`
  - `backend/app/config/env.py:18` — `allowed_origins` default is `http://localhost:5173,http://127.0.0.1:5173`. If an operator forgets to override, a co-located dev server would be a credential-leak vector.
  - **Fix landed:** default empty + startup warning; `.env.example` documents the dev value; `backend/.env` appended with the dev value so the workflow keeps working.
